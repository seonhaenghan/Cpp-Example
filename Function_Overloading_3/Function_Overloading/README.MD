# 함수 오버로딩의 이해 

## 1. C언어에서는 다음과 같이 동일한 이름의 함수가 정의 되는 것을 허용하지 않음 

![image](https://user-images.githubusercontent.com/109562373/203899307-da2eae27-cbc6-4985-bfb8-329af6b69076.png)

## 2. 하지만 C++에서 아래와 같이 Main을 작성했을 때 함수초풀 시 전달되는 인자를 통해서 호출하고자 하는 함수의 구분이 가능하다.
## 이를 Function Overloading이라 한다. 

## 3. C++은 하수 오버로딩을 허용하는데, C언어는 허용하지 않는다. 해당 이유는 호출할 함수를 찾는 방법이 서로 다르기 때문
   >> C++은 호출할 함수를 차자을 때 다음 두 가지 정보를 동시에 활용한다. 
   >> 함수이름, 매개변수의 선언
   >> MyFunc(30, 40);
   >> 두 개의 int형 정수를 인자로 전달 받을 수 있는 MyFunc라는 이름의 함수를 찾는다.

   반면 C언어는 함수의 이름만 이용해서 호출 대상을 찾는다. 
   >> 때문에 C언어에서 함수의 오버로딩이 불가능함, 그리고 이를 문법적으로 허용하지 않음 

## 4. 함수 오버로딩의 예
함수의 오버로딩이 가능하려면 매개변수의 선언이 달라야 한다 예를 들어 다음 두 하수는 오버로딩이 가능하다.

![image](https://user-images.githubusercontent.com/109562373/203900147-dcf80519-25aa-4213-9680-c36e46654a0f.png)

위 와 같은 경우 매개변수 자료형이 다루무로 전달인자의 자료형을 통해서 호출할 함수의 구분이 가능

![image](https://user-images.githubusercontent.com/109562373/203900380-d354c94e-bf1e-4e1d-bfad-2bfd955a0b2f.png)

이 경우는 매개변수의 개수가 다르므로, 전달되는 인자의 개수를 통해 호출할 함수의 구분이 가능

![image](https://user-images.githubusercontent.com/109562373/203900510-d16cfc1e-f620-482f-b265-9d926b30ac35.png)

하지만 이 경우는 두 함수의 반환형이 다르다. 반환형은 함수호출 시 호출되는 함수를 구분하는 기준이 될 수 없다. 
 >> 따라서 위와 같은 형태는 컴파일 에러가 발생한다.

### 정리 >> 함수의 오버로딩을 위한 조건은 매개변수의 자료형이 다르거나 개수가 다르다, 하지만 함수의 반환형이 다른 경우 불가능하다.
